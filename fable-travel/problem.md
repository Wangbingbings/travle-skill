# fable-travel/skill.md 审查问题清单

审查日期：2026-06-30
审查结论：v2.0.0 存在结构性缺陷，需重大修订

---

## 🔴 严重缺陷

### 1. 依赖关系未定义，执行路径是黑盒

**位置：** L33 — `用 superpowers:dispatching-parallel-agents 一次性 spawn 3 个 Agent`

**问题：** 这是整个 pipeline 的核心调用点，但 skill 从未定义：
- 如何向 dispatcher 传递 3 组不同的 instruction + schema
- dispatcher 的返回值格式是什么
- 单个 agent 超时/失败后，其他 agent 的结果怎么处理
- 主进程如何获取 3 个结果以进入 step 3

**影响：** step 2 在当前描述下不可执行。LLM 读到这句话不知道该调什么、传什么参数、拿什么回来。

---

### 2. "NATIVE 模式"是未定义术语

**位置：** L14 — `强制 NATIVE 模式。本 workflow 内联 3 个并行 Agent 完成研究，不依赖外部 Research/ContentAnalysis 技能`

**问题：**
- "NATIVE 模式"不是任何已知框架/平台的术语
- 文件只说了"不是什么"（不依赖外部技能），没说"是什么"
- LLM 无法将"强制 NATIVE 模式"转化为可执行指令

**建议：** 删掉这行或明确定义 NATIVE 模式的执行契约。


---

### 4. 路线合成——最重要的步骤，规格最弱

**位置：** L98-L107

**规格密度对比：**

| 步骤 | 规格行数 |
|------|---------|
| Agent 1 输出格式 | ~20 行 |
| Agent 2 输出格式 | ~12 行 |
| 卡片 B 格式 | ~40 行 |
| **路线合成（核心价值）** | **10 行，5 个要点** |

**问题：** "打地理标签 → 按区域聚合 → 时间排序 → 穿插探店 → 古建融入"这 5 个要点缺少：
- 输出 schema 或模板
- 示例
- 质量判断标准
- 冲突处理方法（如两个推荐的时间冲突）

路线合成是整个 skill 的核心价值主张（从原始研究到可执行行程），当前处理方式使其沦为次要步骤。

### 6. 平台假设链断裂——org-mode + Denote + Windows

**位置：** L166-L168

文件保存路径：`~/Documents/notes/` + Denote naming + `.org`

这条链有 3 个独立假设：

| 假设 | 风险 |
|------|------|
| 用户使用 org-mode | 环境为 win32 + bash，无证据支持 Emacs |
| 用户使用 Denote | 极小众的 Emacs 包，非 Emacs 用户不会用 |
| `~/Documents/notes/` 存在且正确 | Windows 上 bash 的 `~` 解析与预期可能不一致 |

**环境证据：** Claude Code 运行在 win32 平台。生成 `.org` 文件并用 Denote naming，在 Windows 上输出到 `~/Documents/notes/`，既不可测试也不可访问。

## 🟡 值得关注

### 7. "50/50 权重"声明与规格不匹配

**位置：** L3 — `50/50 cultural heritage and local trendy spots`

| 维度 | 规格分量 |
|------|---------|
| Agent 1（文化） | ~30 行详细输出规范，含 3 级子字段嵌套 |
| Agent 2（新潮） | ~15 行较简单的输出 |
| Agent 3 | 探店（吃的）+ 深度内容（混入） |
| org 模板博物馆章节 | 4 级子标题（*** 镇馆之宝 / 重点展厅 / 容易错过的） |
| org 模板新潮章节 | 2 级，无深层字段 |

实际偏向约 70/30 传统文化。应调整规格或修改声明。

---

### 8. 输入解析方式未定义——flag 还是自然语言？

**位置：** L17-L23 和 L28

参数表显示 `-f`（聚焦主题）和 `-n`（天数）是 flag 风格，但 step 1 写"从用户消息中提取城市名称"。

用户应该说 `-f 唐代 -n 4 西安` 还是 `帮我研究西安，4天，聚焦唐代`？两种解析方式行为不同，skill 没选边。

---

### 9. Agent 输出量无目标值

3 个 agent 各产出多少结果完全开放。如 Agent 1 找 15 个博物馆、Agent 2 找 3 家咖啡厅、Agent 3 找 20 家餐厅，合成阶段的决策逻辑是什么？没有"至少 N 个博物馆 / N 条路线 / N 家探店"的量化指导，输出质量随机。
